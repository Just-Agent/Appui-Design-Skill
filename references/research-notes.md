# Research Notes

These notes summarize the reference search used to shape the skill. They are guidance, not copied designs.

## Platform Guidance

- Apple Human Interface Guidelines emphasize hierarchy, harmony, and consistency across platform conventions, plus adaptive layout that respects changing contexts and reading order.
- Apple's layout guidance highlights familiar relationships between controls and content, safe areas, and dynamic adaptation across display sizes and orientations.
- Android's adaptive layout guidance emphasizes designing for changing window sizes, compact/medium/expanded screens, and navigation that adapts beyond phone-only assumptions.
- Material Design 3 navigation bar guidance is a useful baseline for mobile top-level destinations, selected states, and compact bottom navigation conventions.

## Design Inspiration Search

- Current gallery and UI-kit ecosystems such as Awwwards, Mobbin, UI8, Behance, and app-pattern roundups show that useful mobile UI examples are most valuable when they include full flows, real states, and domain-specific content.
- Pattern research is best used for sequencing and hierarchy, not visual cloning. Build original screens around the user's data model and brand personality.
- Strong mobile UI case libraries repeatedly cover finance, health, travel, food, learning, entertainment, home, commerce, care, and social planning because these domains stress different information densities and risk levels.

## Practical Takeaways

- Build example libraries around product archetypes instead of aesthetic trends alone.
- Pair screenshots with source code so the image is reproducible.
- Treat README previews as proof: each image should link back to source and generation workflow.
- Prefer single-purpose screens with realistic content over broad "template packs" that show many shallow placeholders.
